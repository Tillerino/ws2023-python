Klassen

und Objekte

Motivation

  • Zunächst müssen wir uns vorstellen, dass die Programme, die wir schreiben, wesentlich größer sind als die, die wir bisher geschrieben haben.
  • Dies bedeutet, dass wir nicht von zig Zeilen Code ausgehen, sondern von zigtausend Zeilen. Programme können sogar zig Millionen Zeilen Code haben.
  • In dieser Dimension wird die Organisation des Codes ein großes Problem.
  • Üblicherweise arbeiten an Code dieser Größe viele Programmierer gleichzeitig, was seine eigene Herausforderung ist.
  • Funktionen sind ein fundamentales Werkzeug, um Code zu organisieren.
  • Wie man Code in mehrere Dateien aufteilt, werden wir später sehen.
  • Wir werden uns heute anschauen, wie man Daten organisiert und mit Code verbindet.

Beispiel

  • Wir programmieren ein Telefonbuch. Das Telefonbuch hat eine Suchfunktion, die alle Felder gleichzeitig durchsucht.
  • Seien folgende Daten aus einem Schleifendurchlauf über alle Einträge:
    ```cols=40 rows=4 no-out search_term = 'Max' first_name = 'Max' last_name = 'Mustermann' address = 'Musterstraße 1' ```
  • Die Suche wird wie folgt ausgeführt:
    ```cols=40 rows=3 no-out does_match = search_term in first_name \ or search_term in last_name \ or search_term in address ```
    ``` does_match```

Schritt 1: Funktion

  • Da die Suche an mehreren Stellen ausgeführt wird, lagern wir sie in eine Funktion aus.
    ```cols=60 rows=5 no-out def does_match(search_term, first_name, last_name, address): return search_term in first_name \ or search_term in last_name \ or search_term in address ```
    ```cols=60 rows=2 does_match('Max', 'Max', 'Mustermann', 'Müllerstraße 1') ```
  • So können wir die Suchfunktion später verbessern, ohne dass wir alle Stellen, an denen sie verwendet wird, ändern müssen.
  • Beispiele: Wir wollen Groß- und Kleinschreibung ignorieren. Wir wollen mit einem String z.B. "Max Müller" gleichzeitig den Vornamen und die Straße finden. Wir wollen, dass ein leerer String nicht alle Einträge findet 🤦.

Schritt 2: Daten bündeln

  • Wir können mithilfe der Funktion an einer einzigen Stelle verändern, wie die Suche im gesamten Programm ausgeführt wird. 👌
  • Was, wenn wir Notizen im Adressbuch hinzufügen, die jetzt auch durchsucht werden sollen? Wir müssen alle Stellen ändern, an denen die Funktion aufgerufen wird. 🤦
  • Kein Problem: Wir fassen die Daten einfach zusammen:
    ```cols=60 rows=5 no-out def does_match(search_term, entry): return search_term in entry['first_name'] \ or search_term in entry['last_name'] \ or search_term in entry['address'] ```
    ```cols=60 rows=2 entry = {'first_name': 'Max', 'last_name': 'Mustermann', \ 'address': 'Müllerstraße 1'} does_match('Max', entry) ```

Schritt 3: Daten und Funktion zusammenfassen

  • Mithilfe der Funktion und des Dictionaries sind wir nun an einen Punkt, wo wir die Suche vollkommen unabhängig von dem Rest des Programms verändern können. 👌
  • Es bleiben aber ein paar Probleme:
    • Wir haben keine Übersicht, welche Funktionen es gibt, die für einen Telefonbucheintrag geschrieben wurden. Die Gefahr ist Hoch, dass jemand eine zweite, inkonsistente Version der Suche programmiert.
    • Wir haben keine Garantie, dass in die Funktion ein passendes Dictionary übergeben wird: Sind alle Einträge vorhanden? Sind die Einträge gültig? (z.B. Format Telefonnummer).
  • Wir bräuchten:
    1. Eine Garantie, dass nur ein gültiges Dictionary übergeben wird.
    2. Eine Möglichkeit, gültige Dictionaries zu erzeugen und gegen ungültige Veränderungen zu schützen.
    3. Eine Möglichkeit, alle Funktionen, die für einen Telefonbucheintrag geschrieben wurden, zu sammeln.
  • Dies erreichen wir, indem wir Daten und Funktionen mit Klassen zusammenfassen.

Eine tote Klasse

  • Mit `class` können wir etwas Dictionary-ähnliches erzeugen, auf dessen Einträge wir mit einem Punkt zugreifen können.
  • Diese Einträge heißen Attribute oder Felder.
  • Abgesehen von der Notation ist das Ergebnis aber das gleiche wie bei einem Dictionary: Wir können die Attribute wieder lesen.
  • `Entry` ist eine Klasse. `e` ist ein Objekt dieser Klasse.
```cols=35 rows=10 br class Entry: # darf nicht leer sein pass e = Entry() e.first_name = 'Max' e.last_name = 'Mustermann' e.address = 'Müllerstraße 1' print(e.first_name) ```

Eine Methode

  • Interessant wird es, wenn wir Funktionen in Klassen packen.
  • Diese Funktionen heißen Methoden.
  • Im Gegensatz zu normalen Funktionen werden sie mit einem Punkt aufgerufen und bekommen als ersten Parameter das Objekt, auf dem sie aufgerufen werden.
  • Den ersten Parameter nennen wir üblicherweise `self`.
```cols=43 rows=10 br class Entry: def does_match(self, search_term): return search_term in self.first_name \ or search_term in self.last_name \ or search_term in self.address e = Entry() e.first_name = 'Max' e.last_name = 'Mustermann' e.address = 'Müllerstraße 1' print(e.does_match('Max')) ```

Der Konstruktor

  • Es gibt eine spezielle Methode mit dem Namen `__init__`.
  • Diese heißt Konstruktor und wird aufgerufen, wenn ein Objekt erzeugt wird.
  • Der erste Parameter ist das Objekt, das erzeugt wird. Die weiteren Parameter sind die Parameter, die der Konstruktor aufgerufen bekommt.
  • Der Konstruktor wird aufgerufen, wenn der Klassenname mit Klammern aufgerufen wird. Er muss `self` nicht extra zurückgeben.
```cols=46 rows=13 br class Entry: def __init__(self, first_name, last_name, \ address): self.first_name = first_name self.last_name = last_name self.address = address def does_match(self, search_term): return search_term in self.first_name \ or search_term in self.last_name \ or search_term in self.address e = Entry('Max', 'Mustermann', \ 'Müllerstraße 1') print(e.does_match('Max')) ```

Kapselung ("Encapsulation")

  • Häufig ist die Modifikation von Attributen, sodass das Objekt gültig bleibt, komplex.
  • Aus diesem Grund ist es üblich, dass Attribute nur von Methoden derselben Klasse verändert werden können.
  • Wir nennen dies Kapselung ("Encapsulation").
  • In den meisten Programmiersprachen kann man Attribute als private deklarieren, sodass sie nur von Methoden derselben Klasse gelesen und geschrieben werden können.
  • In Python ist dies nicht möglich. Stattdessen gilt die Konvention: Attribute, die mit einem Unterstrich anfangen, sollten nicht von außen gelesen oder geschrieben werden.
```cols=39 rows=18 br no-out class Account: def __init__(self): self._balance = 0 def withdraw(self, amount): if amount <= 0: raise ValueError('Negativ') if amount > self._balance: raise ValueError('Nicht gedeckt') self._balance -= amount def deposit(self, amount): if amount <= 0: raise ValueError('Negativ') self._balance += amount def get_balance(self): return self._balance ```

Objektorientierte Programmierung

Mithilfe von Klassen haben wir unsere Ziele erreicht:
  1. Mit dem Konstruktor (`__init__`) stellen wir sicher, dass nur gültige Objekte erzeugt werden.
  2. Durch Kapselung (Attributnamen mit Unterstrich) stellen wir sicher, dass Objekte nur auf eine gültige Weise verändert werden.
  3. Indem wir Funktionen als Methoden schreiben, stellen wir sicher, dass sie nur auf gültige Objekte angewendet werden können.
  4. Alle Methoden, die für eine Klasse relevant sind, sind in der Klasse zusammengefasst.
  • Klassen sind ein Konzept aus der objektorientierten Programmierung. In der objektorientierten Programmierung werden Programme als eine Interaktion zwischen selbstverwalteten Daten modelliert.
  • Die objektorientierte Programmierung ist eines von mehreren architektonischen Konzepten, die in der Softwareentwicklung verwendet werden.

Klasse und Objekt

  • Die Klasse (rechts `C`) ist eine Art Bauplan für Objekte. Sie enthält die Methoden, die Objekte haben sollen (rechts keine).
  • Ein Objekt (rechts `o`) ist eine Instanz einer Klasse. Es besteht aus den konkreten Werten der Attribute (rechts keine).
```cols=15 rows=5 br no-out class C: # leer pass o = C() ```
  • Die Klasse selber ist ein Objekt.
    ``` print(C) ```
  • Die Klasse einer Instanz kann man mit der Funktion `type` bekommen.
    ``` print(type(o)) ```
  • Diese beiden Ausdrücke sind hier äquivalent:
    ``` C == type(o) ```
  • So kann man z.B. prüfen, ob ein Objekt eine Instanz einer Klasse ist.

Kleingedrucktes

```cols=26 rows=10 br class C: def __init__(self): self.x = 1 def noise(self): print('noise', self.x) o = C() o.x = 123 o.noise() ```
  • Methoden sind eigentlich nur Funktionen im namespace der Klasse. Durch verwendung des Punktes wird der Ausdruck vor dem Punkt zum ersten Argument des Funktionsaufrufs. Folgendes ist also äquivalent:
    ``` C.noise(o) ```
  • Das erklärt auch, warum wir `self` als ersten Parameter von Methoden brauchen. Python ist eine sehr kleine Sprache. Ebenso wie mit private feldern, ist hier kein wirkliche Funktionalität, sondern nur eine Konvention.
  • Wir haben schon gesehen, dass die Klasse über `type` bestimmt werden kann. Folgendes ist also ebenfalls äquivalent.
    ``` type(o).noise(o) ```

Zusammenfassung

  • Klassen sind eine Möglichkeit, Daten und Funktionen zusammenzufassen.
  • Die Daten heißen Attribute oder Felder.
  • Die Funktionen heißen Methoden.
  • Klassen werden mit dem `class` Schlüsselwort definiert.
  • Auf Attribute und Methoden wird mit einem Punkt zugegriffen.
  • Methoden bekommen als ersten Parameter das Objekt, auf dem sie aufgerufen werden. Diesen nennt man per Konvention `self`.
  • Objekte werden mit dem Klassennamen und Klammern erzeugt.
  • Dabei wird eine spezielle Methode mit dem Namen `__init__` aufgerufen.
```cols=34 rows=13 br no-out class Account: def __init__(self, initial=0): if initial < 0: raise ValueError('Negativ') self._balance = initial def deposit(self, amount): if amount <= 0: raise ValueError('Negativ') self._balance += amount def get_balance(self): return self._balance ```
```cols=34 rows=3 br a = Account(50) a.deposit(100) a.get_balance() ```

Fließkommazahlen

Fließkommazahlen

  • Bisher haben wir nur mit ganzen Zahlen gearbeitet.
  • In Python gibt es auch Kommazahlen, genauer: Fließkommazahlen (engl. floating point number, in Python `float`). Das Komma wird englisch notiert, also als Punkt: `1.5`.
  • Sie lassen sich grundsätzlich genau wie ganze Zahlen verwenden:
    ```cols=18 1.5 + float('2.3') ```
    ```cols=28 br 1.5 ** 4 / (23.4 - 2.3) ```
  • Fließkommazahlen können mit einem Exponenten geschrieben werden - der sogenannten wissenschaftlichen Notation:
    ```cols=7 1.5e12 ```
    ```cols=7 1.5e-4 ```
  • Das Ergebnis einer ganzzahligen Division ist immer eine Fließkommazahl:
    ```cols=15 5 / 2 ```
    im Gegensatz zu der Division mit dem `//` Operator:
    ```cols=15 5 // 2 ```.
  • Operationen, die ganze Zahlen und Fließkommazahlen mischen, geben immer eine Fließkommazahl zurück:
    ```cols=15 5.5 / 2.5 ```
    ```cols=15 5 + 2.5 ```
  • Auch, wenn das Ergebnis ganzzahlig Zahl ist:
    ```cols=15 5.0 * 2.0 ```
    ```cols=15 br type(5.0 * 2.0) ```

Fließkommazahlen Warnung

  • Wie genau Fließkommazahlen funktionieren, ist kein Thema für diesen Kurs. Sie sollten aber ein paar Beispiele sehen um zu verstehen, dass sie weder reelle noch rationale Zahlen sind:
  • ```cols=40 fragment-out 0.1 + 0.2 ```
  • ```cols=40 fragment-out 1 + 0.0000000000000001 ```
  • ```cols=40 fragment-out 1 - 0.0000000000000001 ```
  • ```cols=40 fragment-out 1E308 * 2 ```
  • ```cols=40 fragment-out 1E308 * 2 / (1E308 * 2) ```
  • ```cols=40 fragment-out 1E-308 / 2 ```
  • ```cols=40 fragment-out 1E-308 * 0.0000000000000001 ```
  • Fließkommazahlen sind nützlich um näherungsweise zu rechnen, aber nicht um exakt zu rechnen.
  • Wenn Sie z.B. mit Geld rechnen, ist es besser in ganzen Cent zu rechnen, als in Euro mit Komma.

Übung II: Auto

  • Definieren Sie eine Klasse `Car`.
  • Objekte dieser Klasse sollen einen Kilometerzähler (in km), einen Tankfüllstand (in l) und einen Verbrauch (in l/100km) enthalten.
  • Der Kilometerzähler und der Tankfüllstand sollen mit 0 initialisiert werden. Der Verbrauch soll als Parameter im Konstruktor übergeben werden.
  • Die Klasse soll eine Methode `report` enthalten, die den Kilometerzähler und den Tankfüllstand ausgibt.
  • Die Klasse soll eine Methode `drive` enthalten, die einen Parameter `km` bekommt und den Kilometerzähler und den Tankfüllstand entsprechend verändert. Wenn der Tankfüllstand nicht ausreicht, um die Strecke zu fahren, soll das Auto so weit fahren, wie es kann und dann stehen bleiben. Eine entsprechende Meldung soll ausgeben werden (kein Fehler geworfen).
  • Letzendlich soll die Klasse eine Methode `refuel` enthalten, die den Tank auffüllt. Der Parameter ist die Menge (in l), die getankt werden soll.
  • Beachten Sie, dass Sie beim berechnen des Verbrauchs eine Division verwenden und deshalb eine Fließkommazahl erhalten. Dies führt dazu, dass der Kilometerzähler und der Tankfüllstand auch Fließkommazahlen werden. Trotz der vorherigen Warnungen ist dies hier in Ordnung.
```hidden class Car: def __init__(self, consumption): self._consumption = consumption self._km = 0 self._fuel = 0 def report(self): print('km:', self._km, 'Tank (l):', self._fuel) def drive(self, km): fuel_needed = km / 100 * self._consumption if fuel_needed > self._fuel: km = self._fuel / self._consumption * 100 self._fuel = 0 print('Tank leer :(') else: self._fuel -= fuel_needed self._km += km def refuel(self, amount): self._fuel += amount ``` Hier ist ein Beispiel, wie sich die Klasse verhalten würde:
```cols=25 rows=10 br car = Car(5) car.drive(100) car.refuel(10) car.report() car.drive(100) car.report() car.drive(150) car.report() ```
