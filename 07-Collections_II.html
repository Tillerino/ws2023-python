Tupel

Tupel

  • Ein Tupel ist eine unveränderbare Liste. Es wird mit Klammern definiert und ist sonst ähnlich verwendbar:
    ```cols=22 t = (5, 6, 7, 8, 9) t[1] ```
    ```cols=22 t[1:3] ```
  • Versucht man ein Tupel zu verändern, gibt es einen Fehler:
    ```cols=22 t[1] = 1 ```
  • Das gleiche gilt für die anderen modifizierenden Methoden wie `append` oder `pop`.

Warum Tupel?

1. Tupel verkörpern eine andere Idee als Listen.
  • Listen werden üblicherweise für eine variable Anzahl von gleichartigen Elementen verwendet.
    • Die Anzahl der Elemente steht beim Programmieren noch nicht fest oder verändert sich im Laufe des Programms.
    • Dies ist typischerweise eine Eingabe des Programms.
  • Tupel werden üblicherweise für eine konstante Anzahl von ggf. verschiedenartigen Elementen verwendet.
    • Die Anzahl der Elemente steht beim Programmieren schon fest.
    • Die Komponenten eines Tupels sind unterschiedliche Eigenschaften desselben Objekts.
2. Tupel können gehasht werden.
  • Da sie nicht veränderbar sind, implementiert Python eine Hashfunktion für Tupel.
  • Sie können daher z.B. als Schlüssel in einem Dictionary verwendet werden:
    ```cols=30 d = { (1, 'a'): [ 1, 2, 3 ] } d[(1, 'a')] ```

Entpacken

  • Tupel können auch auf der linken Seite einer Zuweisung stehen. In diesem Fall werden die werte entpackt
    ```cols=15 t = (1, 2, 3) (a, b, c) = t a ```
    ```cols=15 b ```
    ```cols=15 c ```
  • Zusätzlich können die Klammern sowohl beim Erstellen als auch beim Entpacken eines Tupels weggelassen werden:
    ```cols=15 t = 1, 2, 3 a, b, c = t a ```
  • Tatsächlich kann eine Funktion nicht mehrere Werte zurückgeben. Mit Tupeln sieht es nur so aus:
    ```cols=20 def f(): return 1, 2 a, b = f() b ```
  • Das Gleiche gilt für das Iterieren über Schlüssel-Wert-Paare eines Dictionarys. Hier werden tatsächlich Tupel erstellt und entpackt:
    ```cols=25 d = { 'a': 1, 'b': 2 } for k, v in d.items(): print(k, v) ```

Mengen

Mengen

  • Eine Menge ist eine ungeordnete Sammlung von Elementen, in der jedes Element nur einmal vorkommt. Sie wird mit geschweiften Klammern definiert:
    ```cols=20 s = { 2, 1, 3, 2 } s ```
  • Ungeordnet bedeutet hier, dass die Elemente in einer Menge nicht unbedingt ihre Reihenfolge behalten.
  • Eine Menge wendet wie ein Dictionary eine Hashfunktion auf ihre Elemente an. Dies bedeutet, dass das Prüfen, ob ein Element in einer Menge vorkommt, sehr schnell geht:
    ```cols=20 2 in s ```
  • Dies bedeutet aber auch, dass es die gleichen Einschränkungen wie bei Dictionary-Schlüsseln gibt:
    ```cols=25 s = { [1, 2], ['x'] } ```
  • Die Hashfunktion ist auch der Grund, warum die Menge oben sortiert aussieht. Mengen sind aber grundsätzlich auch nicht sortiert:
    ```data-ns=new br s = { 41, 12, 9 } s ```

Modifikation

  • Wie bei Listen gibt es Methoden, um Mengen zu verändern. Die Methoden heißen etwas anders:
    ```cols=20 s = { 1, 2, 3 } s.add(4) s ```
  • ```cols=20 s.update({ 5, 6 }) s ```
    `update` akzeptiert auch Listen oder Tupel.
  • ```cols=20 s.remove(2) s ```

Comprehensions

Comprehensions

  • Folgenden Code werden Sie häufig sehen:
    ```cols=23 br l1 = [ 1, 2, 3, 4, 5 ] l2 = [] for x in l1: l2.append(x*x) l2 ```
  • Dieser ist mit einer List Comprehension wesentlich kompakter:
    ```cols=27 br l2 = [ x*x for x in l1 ] l2 ```

Filter

  • Schleife:
    ```cols=23 br l1 = [ 3, 2, 4, 2, 1 ] l2 = [] for x in l1: if x > 1: l2.append(x*x) l2 ```
  • So wie in einer Schleife kann man auch in einer List Comprehension filtern.
  • Comprehension:
    ```cols=31 br l2 = [x*x for x in l1 if x > 1] l2 ```

Tupel, Mengen, Dictionaries

  • Es gibt für alle vorgestellten Collections Comprehensions. Listen:
    ```cols=45 l1 = [ 3, 2, 4, 2, 1 ] l2 = [ x*x for x in l1 if x > 1 ] l2 ```
  • Tupel:
    ```cols=45 t = tuple(x*x for x in l1 if x > 1) t ```
  • Mengen:
    ```cols=45 s = { x*x for x in l1 if x > 1 } s ```
  • Dictionaries:
    ```cols=45 d = { x: x*x for x in l1 if x > 1 } d ```

Vertiefung: Generatorausdrücke

  • Warum mussten wir eigentlich `tuple(...)` schreiben und nicht einfach nur `(...)`?
  • ```cols=30 br l = [ 3, 2, 4, 2, 1 ] t = (x*x for x in l if x > 1) t ```
  • Die Klammern sind hier reguläre Klammern und haben nichts mit Tupeln zu tun. Der Ausdruck in den Klammern ist ein Generatorausdruck.
  • Überall, wo "Iterierbares" (bisher Listen, Tupel, Mengen oder `range`) stehen kann, können auch Generatorausdrücke stehen:
    ```cols=30 sum(x*x for x in l if x > 1) ```
  • ```cols=40 for x in (x*x for x in l if x > 2): print(x+5) ```
  • Ja, auch in Comprehensions/Generatorausdrücken:
    ```cols=45 [ x+5 for x in (x*x for x in l if x > 2) ] ```

  • Ein Anagramm ist ein Wort, das aus den Buchstaben eines anderen Wortes durch Umordnen gebildet werden kann.
  • Beispiel: "Ergbut" ist ein Anagramm von "Betrug".

  • Wichtig: die Häufigkeit der Buchstaben muss dabei erhalten bleiben.

Pickle

  • `pickle` ist ein Python-Modul, das Objekte serialisiert und deserialisiert. Es schreibt z.B. ganze Datenstrukturen in eine Datei oder liest sie aus.
  • Dies ist zwar nicht notwendig, aber damit machen die Übungen mehr Spaß.
  • Rechts ist ein Beispiel über die Verwendung von `pickle`.
  • Was genau `with`, `try` und `except` tun, besprechen wir später.
  • Kopieren Sie den Inhalt der oberen Box und verwenden Sie die Funktionen `save` und `load` in Ihren Übungen, wenn Sie möchten.
  • Das zweite Argument von `load` ist der Standardwert, falls die Datei nicht existiert.
  • Die untere Box zeigt, wie Sie die Funktionen verwenden können.
  • Führen Sie die untere Box mehrmals aus um den Effekt des Speicherns zu sehen.
import pickle def save(file, data): with open(file, 'wb') as f: pickle.dump(data, f) def load(file, default): try: with open(file, 'rb') as f: return pickle.load(f) except FileNotFoundError: return default

Übung II: Personenverzeichnis

  • Sie müssen das Personenverzeichnis der Republik verwalten.
  • Das Verzeichnis soll die Namen aller Personen enthalten, die in der Republik leben.
  • Leider sind die Namen nicht einzigartig, da es z.B. mehrere Personen mit dem Namen "Susanne Müller" gibt.
  • Mit dem Personenverzeichnisinnovationsgesetz von 2023 wurde allerdings beschlossen, dass jeder eine kleine Zusatznummer bekommt, mit der der Name dann eindeutig wird. Somit kann man nun Susanne Müller #14 und Susanne Müller #39 unterscheiden.
  • Einmal vergeben, kann die Zusatznummer nicht mehr geändert werden. Als Identifikation für diese Person gilt dann der Name und die Zusatznummer.
  • Die Zusatznummer muss eine positive ganze Zahl, eher klein (das ist Ermessenssache) - und natürlich noch nicht vergeben sein. Ansonsten ist die Wahl ganz ihnen überlassen und Sie möchten gerne Wünschen entgegenkommen.
  • Mir wird niemand Tillmann Gaida #1 streitig machen, aber bei anderen Namen gilt es sicher zu gehen. Michael Schmidt #69 kann schließlich nur einer sein.
  • Da Sie es können, schreiben Sie ein Programm um Ihre Arbeit zu erleichtern. Das Programm soll folgende Funktionalität bieten:
    1. Das gesamte Verzeichnis auflisten. Jeder Name soll zusammen mit der Zusatznummer ausgegeben werden.
    2. Personen hinzufügen. Dabei soll ein Name und eine Zusatznummer angegeben werden. Ist die Kombination bereits vergeben, soll es eine Fehlermeldung geben. Außerdem soll geprüft werden, ob die Zusatznummer positiv ist.
    3. Kombination prüfen. Dabei soll eine Kombination aus Name und Zusatznummer angegeben werden. Das Programm soll ausgeben, ob die Kombination vergeben ist.
